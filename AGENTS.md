# Agent Notes for vnstock

## Setup And Commands
- Python package, `requires-python >=3.10`; install local dev tooling with `pip install -e ".[dev]"` or test-only tooling with `pip install -e ".[test]"`.
- Reproducible install (matches CI): `pip install -r requirements.lock && pip install -e . --no-deps`.
- Required local verification is `make verify`; it runs `ruff check --fix .`, `ruff format .`, `ruff check .`, `pre-commit run --all-files`, then pytest.
- Faster focused checks: `make lint`, `make format`, `make pre-commit`, `PYTHONPATH=. pytest tests/unit/api/test_quote.py`, or `PYTHONPATH=. pytest tests/unified_ui/test_registry.py`.
- `make test` first runs `PYTHONPATH=. pytest tests/unified_ui/`, then `PYTHONPATH=. pytest tests/`; expect `tests/unified_ui` to run twice — this is intentional.
- CI subset (fast, no network): `PYTHONPATH=. pytest -m 'not slow' tests/unit/core tests/unit/ui tests/unified_ui tests/contracts`.
- `pytest.ini` is the active pytest config (takes precedence over `pyproject.toml [tool.pytest.ini_options]`): strict markers/config, verbose output by default, 300s timeout. Always use `PYTHONPATH=.` for local test runs to force the working-tree package.
- Live tests are gated: `VNSTOCK_LIVE_TESTS=true PYTHONPATH=. pytest tests/live/providers -m live`. Target specific providers with `VNSTOCK_LIVE_PROVIDERS=DNSE` and symbols with `VNSTOCK_LIVE_SYMBOLS=FPT`.
- `pyproject.toml` has `--ignore-glob='*tcbs*'` in `addopts`, which suppresses tcbs tests by default in some run modes.

## Architecture
- Public package entrypoint is `vnstock/__init__.py`; it exports legacy API classes (`Quote`, `Listing`, `Company`, `Finance`, `Trading`, `Vnstock`) plus v4 Unified UI (`Reference`, `Market`, `Fundamental`, `Retail`). `Broker`, `show_api`, `show_doc`, visualization (`Chart`/`.viz`), and bot/notification (`Messenger`) have been removed.
- New user-facing APIs should go through the Unified UI under `vnstock/ui`. Top-level domain objects create subdomain objects; leaf methods route through `BaseUI._dispatch()` using `vnstock/ui/_registry.py::MAP`.
- Keep `vnstock.ui` as facade/routing only. Put data extraction in providers, not UI methods. Scope is data extraction and normalization only — no charting, no notifications, no broker execution.
- Provider split: `vnstock/explorer/{kbs,vci,msn,fmarket,misc,dnse}` for scraping/public web data; `vnstock/connector/fmp` for the FMP data API connector; `vnstock/api/*` are legacy adapters built on `vnstock/base.py::BaseAdapter`. The `vnstock/connector/dnse` broker connector has been removed; DNSE market data lives in `vnstock/explorer/dnse`.
- Provider modules self-register at import time with `vnstock.core.registry.ProviderRegistry.register(provider_type, source, Class)`. Add providers by registration + MAP entries, not UI `if/else` source branching. Also add a `POOLS` entry in `vnstock/ui/_pools.py` when the new provider supports an existing multi-provider method.
- **Two registries — do not confuse them.** `vnstock/core/registry.py::ProviderRegistry` is the active runtime registry used by `vnstock/base.py` and UI dispatch. `vnstock/core/base/registry.py` is a separate decorator-style registry.
- `BaseUI._dispatch()` auto-selects providers via `vnstock/core/router.py::ProviderRouter` (round-robin + cooldown). Multi-provider pools are declared in `vnstock/ui/_pools.py::POOLS`. Callers that pass `source=` explicitly bypass the router.
- **Cache layer** (`vnstock/core/cache.py::CacheManager`): disabled by default at the `_dispatch` level even though `CacheConfig.enabled=True`. Enable via `VNSTOCK_CACHE_ENABLED=true` or `use_cache=True` kwarg. Config in `vnstock/core/settings.py::CacheConfig`.
- **Quality validation layer** (`vnstock/core/quality/`): off by default (`QualityConfig.enabled=False`). Enable per-call with `validate=True` or globally with `VNSTOCK_QUALITY_ENABLED=true`. Quality report attaches to `DataFrame.attrs['quality']`.

## Repo-Specific Gotchas
- Flat access model: do not add package-level user registration, tier gates, entitlement checks, or private-package fallback routing. Keep external provider credentials scoped to their connector/provider.
- UI modules intentionally use lazy runtime imports and `TYPE_CHECKING` blocks to avoid circular imports. `vnstock/__init__.py` uses a lazy proxy for the `Vnstock` class. Preserve that pattern when adding domain accessors or type hints.
- Ruff is the local formatter/linter source of truth: line length 88, target `py310`, double quotes, lint families `E/W/F/I/B/C4` with `E501` ignored. `.flake8` exists as a legacy artifact only.
- Do not add third-party dependencies unless explicitly approved. `requirements.lock` (generated by `uv pip compile`) and `uv.lock` are present; do not edit them manually.
- Repo-local OpenSpec/OpenCode workflow exists under `.opencode/commands/opsx-*.md` and `openspec/`; use `/opsx-propose`, `/opsx-apply`, `/opsx-explore`, or `/opsx-archive` only when the user asks for that workflow.
